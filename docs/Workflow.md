## 1. 변경이력 관리 (필수)

### 1.1 개요

모든 스크립트의 변경이력은 **별도의 history 파일**로 관리한다.

### 1.2 폴더 구조

```
(repo root)
├── history/
│   ├── {스크립트명}_history.md
│   └── ...
├── scripts/
│   └── ...
└── docs/
```

### 1.3 파일명 규칙

```
{스크립트명}_history.md
```

**예시:**
- `Apache_Linux_history.md` - Apache_Linux.sh의 변경이력
- `MySQL_Linux_history.md` - MySQL_Linux.sh의 변경이력
- `PC_Windows_history.md` - PC_Windows.ps1의 변경이력

### 1.4 버전 넘버링

```
YYMMDDHH[a-Z]
```

| 구성 | 설명 | 예시 |
|------|------|------|
| `YY` | 연도 (2자리) | 25, 26 |
| `MM` | 월 (2자리) | 01, 12 |
| `DD` | 일 (2자리) | 01, 31 |
| `HH` | 시 (2자리, 24시간) | 00, 23 |
| `a-Z` | 수동 구분자 (선택) | a, b, c |

**예시:**
- `25010914` - 2025년 1월 9일 14시 (자동 생성)
- `25010914a` - 같은 시간대 첫 번째 수동 구분
- `25010914b` - 같은 시간대 두 번째 수동 구분

### 1.5 History 파일 형식

```markdown
# {스크립트명} 변경이력

## {버전} | {작성자}
- 변경 내용 1
- 변경 내용 2

## {버전} | {작성자}
- 최초 작성
```

**예시:**
```markdown
# Apache_Linux.sh 변경이력

## 26010914 | Seedgen
- 전체 항목 포함 규칙 적용
- N/A 항목 추가 (WEB-01, 02, 03, 13, 15, 23)

## 25061814 | Seedgen
- 최초 작성
```

### 1.6 작성자 표기

| 구분 | 표기 예시 |
|------|----------|
| 개인 작성 | 실명 또는 팀명 (`홍길동`, `Seedgen`) |
| AI 작성 | 모델명 명시 (`Seedgen`, `ChatGPT`) |

### 1.7 기록 시점

다음 상황에서 반드시 history 파일을 업데이트한다:

1. 스크립트 최초 작성
2. 진단 항목 추가/수정/삭제
3. 버그 수정
4. 구조 변경

## 2. 이슈 처리 루틴

### 2.1 개요

테스트 환경에서 발견된 버그, 개선사항 등을 체계적으로 처리하는 절차를 정의한다.

### 2.2 이슈 리포트 형식

```
환경 정보

OS : {운영체제} {서비스 버전}
스크립트 버전 : {버전} ({날짜} 자)
--
항목코드	항목명	비고
{코드}	{항목명}	- {이슈 내용}
{코드}	{항목명}	- {이슈 내용}
...
```

**예시:**
```
환경 정보

OS : windowserver 2022 Microsoft SQL Server 2025 (RTM) - 17.0.1000.7
스크립트 버전 : v26010916e(26-01-09 자)
--
항목코드	항목명	비고
D-01	기본 계정의 비밀번호	- sa 계정이 존재하지만 조회되지 않음
D-03	비밀번호의 사용기간	- 복잡도 및 만료 설정 모두 확인 필요
```

### 2.3 이슈 처리 프로세스

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 이슈 접수                                                    │
│     └─ 이슈 리포트 확인                                          │
├─────────────────────────────────────────────────────────────────┤
│  2. 이슈 검증                                                    │
│     ├─ 유효: 버그 또는 개선 사항으로 확인됨 → 수정 진행          │
│     └─ 기각: 이미 처리됨, 동작이 정상임, 테스터 오류 등          │
├─────────────────────────────────────────────────────────────────┤
│  3. 수정 작업                                                    │
│     ├─ 스크립트 수정                                             │
│     └─ 테스트 (가능한 경우)                                      │
├─────────────────────────────────────────────────────────────────┤
│  4. History 업데이트                                             │
│     └─ history/{스크립트명}_history.md 업데이트                  │
├─────────────────────────────────────────────────────────────────┤
│  5. 버전 부여                                                    │
│     └─ YYMMDDHH[e] 형식으로 새 버전 할당                         │
└─────────────────────────────────────────────────────────────────┘
```

### 2.4 이슈 검증

#### 검증 기준

| 판정 | 기준 | 처리 |
|------|------|------|
| **유효** | 실제 버그 또는 개선이 필요한 사항 | 수정 진행 |
| **기각** | 이미 수정됨, 정상 동작, 테스터 오해 | 사유 기록 후 스킵 |
| **보류** | 추가 정보 필요, 재현 불가 | 테스터에게 확인 요청 |

#### 검증 방법

1. **코드 확인**: 해당 항목 함수의 현재 로직 검토
2. **비교 분석**: 이슈 내용과 실제 코드 동작 비교
3. **히스토리 확인**: 이전에 동일 이슈가 처리되었는지 확인

### 2.5 버전 넘버링 (확장)

기본 버전 형식에 **편집 구분자** 추가:

```
YYMMDDHH[e]
```

| 구성 | 설명 | 예시 |
|------|------|------|
| `YY` | 연도 (2자리) | 26 |
| `MM` | 월 (2자리) | 01 |
| `DD` | 일 (2자리) | 15 |
| `HH` | 시 (2자리) | 16 |
| `e` | 편집 마커 (이슈 수정 시) | e |

**예시:**
- `26011516e` - 2026년 1월 15일 16시, 이슈 수정 버전

### 2.6 History 기록 형식

이슈 수정 시 history 파일에 다음 형식으로 기록:

```markdown
## {버전} | {작성자}
- {항목코드}: {수정 내용 요약}
  - 상세 변경 사항 1
  - 상세 변경 사항 2
```

**예시:**
```markdown
## 26011516e | Seedgen
- D-03: 비밀번호 정책(is_policy_checked)과 만료 정책(is_expiration_checked) 동시 확인
  - Windows 암호 정책(net accounts) 출력 추가
  - 양호 조건: 두 정책 모두 적용 시
- D-11: PUBLIC/guest sys 객체 접근 권한 탐지 로직 전면 수정
  - 시스템 DB별(master, msdb, model, tempdb) 권한 조회
  - sys 스키마 객체 GRANT 권한 정확히 탐지
```

### 2.7 수정 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **버그** | 잘못된 판정, 오탐, 미탐 | "PUBLIC 권한이 탐지되지 않음" |
| **개선** | 출력 정보 부족, UX 개선 | "양호해도 결과값 출력 필요" |
| **오류** | 스크립트 실행 오류 | "XML 파싱 오류 발생" |
